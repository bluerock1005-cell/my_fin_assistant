## 对齐图片规范重构（同日第二轮）

### 背景
用户提供第二张规范截图，明确 4 条设计规则：
1. 主窗口入口做"壳"，不写任何业务逻辑；侧边栏按钮用 QStackedWidget 切换，点就 setCurrentWidget()
2. **每个功能模块暴露统一接口** FeatureModule，主窗口靠 get_widget 加载，**不用互相 import 细节**
3. **主窗口用 FEATURES 注册表统一装载**，加新功能只需加一行，不用改主窗口逻辑
4. **UI 和逻辑严格分离**：ui.py 只管界面交互，logic.py 放纯计算数据处理，方便单独测试或替换

### 重构改动清单

**core/feature_base.py — 全量重写**
- 类名 `BaseFeature` → **`FeatureModule`**
- 方法 `build_widget(self)` → **`get_widget(self, parent=None)`**
- icon 类型 `Optional[QIcon]` → **`str`**（图标路径字符串）
- 去掉 `display_name`、`on_activate/on_deactivate`、懒加载 widget 缓存等不在规范中的接口
- 接口极简：只留 `name: str` / `icon: str` / `get_widget(parent) -> QWidget`

**core/main_window.py — 重写**
- import 改用 **`FeatureModule`**
- 去掉占位页 hack（stack 不再预留索引0），改用 `_widgets: dict[int, QWidget]` 懒加载映射
- `register_feature()` 不再立即创建 widget，只在首次切换时调 `feat.get_widget(parent=self._stack)`
- icon 按 str 路径加载 QIcon（优先从 `app_config.ICONS_DIR` 解析）
- 切换用 **`setCurrentWidget()`**

**features/bank_classify/ui.py — tkinter → Qt 全量重写**
- 原 `BankGuiApp(tk.Tk)` → 新 `BankClassifyFeature(FeatureModule)` + `BankClassifyWidget(QWidget)`
- UI 功能完整保留：剪贴板粘贴、文件加载（xlsx/csv/txt）、输出路径浏览、生成 Excel、清空
- 调用层不变：仍走 `classify_logic.load_banks()` / `build_workbook()`，纯逻辑零修改
- 按规范拆两层：`BankClassifyFeature` 是模块入口（给主窗口调 get_widget），`BankClassifyWidget` 是具体控件

**features/home/feature.py — 对齐 FeatureModule**
- `HomeFeature(BaseFeature)` → **`HomeFeature(FeatureModule)`**
- `build_widget()` → **`get_widget(parent=None)`**
- 抽出 `HomeWidget(QWidget)` 作为具体控件类

**main.py — FEATURES 注册表模式**
- `load_features()` 函数 → **`FEATURES: list = []` 常量列表**
- 每个功能一行 try/except 实例化，注释掉即不加载互不影响
- 已注册：`HomeFeature()`、`BankClassifyFeature()`
- 预留 `InvoiceFeature` 注释模板
